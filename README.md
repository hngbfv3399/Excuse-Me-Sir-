# 🏃‍♂️ Excuse Me Sir (멀티플레이어 실시간 추격전 게임)

## 📖 프로젝트 개요
**Excuse Me Sir**는 웹 소켓(Socket.io) 기반의 실시간 온라인 멀티플레이 팀 대항전 게임입니다. 플레이어들은 '의적(도망자)'과 '관군(술래)' 팀으로 나뉘어 한정된 맵 안에서 금괴를 탈취하거나 이를 저지하기 위해 추격전을 벌입니다.

## ✨ 프로젝트 프로세스 및 특징
* **실시간 멀티플레이어 동기화:** Node.js와 `Socket.io`를 활용하여 실시간으로 여러 명의 유저가 지연 없이 하나의 게임 룸에 모이고 서로의 좌표와 맵 상황을 연동합니다.
* **클린 아키텍처 및 철저한 역할 분리:**
  * **순수 렌더러 JSX:** 리액트 컴포넌트(`GameCanvas.jsx` 등)는 내부적으로 어떠한 복잡한 연산도 하지 않고, 상태 값을 화면상에 뿌려주는 프레젠테이션(Presentation) 역할만 철저히 수행하도록 분리되어 있습니다.
  * **Custom Hooks 기반 로직 추상화:** 방대한 네트워크 이벤트 처리와 키보드/조이스틱 기반의 복잡한 물리엔진 충돌/계산 로직이 `useGameState`와 `usePlayerPhysics` 등의 Custom 훅으로 완벽히 분리되어 있어 가독성이 탁월합니다.
  * **순수 데이터 기반(Data-Driven) 기획:** 캐릭터의 이동 속도, 직업 스킬, 팀별 전용 아이템 배분 로직 등 게임 내 모든 기획적 기믹이 기능 코드에 하드코딩되지 않고 `gameData` 폴더의 순수 객체 형태로 정의되어 강력한 확장성을 가집니다.
* **고성능 WebGL 렌더링 적용:** Canvas 2D 방식에서 벗어나 `PixiJS` 그래픽 엔진을 도입함으로써 넓은 배경 타일맵 렌더링, 스프라이트 레이어 분리 및 로드 퍼포먼스를 브라우저 환경에서 극대화했습니다.

---

## 📂 프로젝트 폴더 구조 및 각 폴더별 역할

### 클라이언트 영역 (`/client`)
프론트엔드 환경을 담당하며 React 프레임워크와 Vite, PixiJS가 결합되어 있습니다.

* **`/src/components/`**
  - 리액트 **시각적 UI 레이어** 폴더입니다.
  - `GameCanvas.jsx`: 분리된 훅들로부터 로직 데이터를 받아 PixiJS 렌더러와 동기화시킵니다.
  - `WaitingRoom.jsx`: 팀 구분, 소켓 연결 상태 표시 등의 대기실 화면입니다.
  - `MobileControls.jsx`: 모바일 환경을 위한 가상 조이스틱과 아이템 사용 UI를 출력합니다.
* **`/src/renderer/`**
  - 눈에 보이는 **픽셀 아트와 WebGL 그래픽 렌더링**을 전담하는 순수 PixiJS 모듈입니다.
  - `engine.js`: App Stage 관리, 뷰포트 센터링 및 화면 주사율 동기화 제어.
  - `MapLayer.js`: `constants/map.js` 데이터를 해석해 배경과 벽의 타일시트를 매핑합니다.
  - `EntityLayer.js`: 동적 요소인 캐릭터, 감옥 오브젝트, 필드 금괴 아이템 이미지를 표시합니다.
* **`/src/hooks/`**
  - 무거운 비즈니스 연산 로직이 담긴 **React Custom Hook** 폴더입니다. 기능코드는 기능 폴더에 있어야 한다는 원칙이 적용된 곳입니다.
  - `useGameState.js`: 서버로부터 전송되는 글로벌 게임 데이터(스코어, 잔여시간, 페이즈 전환) 관련 소켓 이벤트를 전담 관리합니다.
  - `usePlayerPhysics.js`: 매 프레임마다 내 캐릭터가 키보드나 조이스틱 지시에 따라 바라보는 방향의 다음 좌표(X, Y)를 계산하고 충돌(`isWall`)을 검증한 뒤에만 서버로 패킷을 전송하는 물리 통제소입니다.
* **`/src/gameData/`**
  - 오로지 기획 데이터만 모아둔 **데이터 딕셔너리** 폴더입니다. 기능적인 로직 없이 값을 반환하는 사전 역할을 합니다.
  - `characters.js`: 의적, 관군 등 각 캐릭터의 이름, 기본 속도 상수 명세 공간.
  - `items.js`, `skills.js`: 타입명, 스프라이트 이미지의 `tx`, `ty` 타일 인덱스 등을 정의합니다.
* **`/src/constants/`**
  - 변하지 않는 기하학적 상수, 2D 배열(`MAP` 타일 행렬) 값을 관리합니다.
* **`/src/utils/`**
  - 클라이언트에서 맵 타일 배열의 특수 범위(벽, 세이프존 등)를 계산해 주는 단순 유틸리티 함수 모음입니다.

### 서버 영역 (`/server`)
게임의 공통 상태를 중재하는 Node.js 기반 실시간 이벤트 처리기입니다.

* **`/index.js`**
  - 웹서버 인스턴스를 열고 전체 클라이언트 접속을 통제하는 서버 코어 파일입니다.
  - 룸 매칭, 팀 강제 분배 시스템(`joinRoom`), 도망자가 금괴 획득 시 득점 처리 로직, 술래에게 닿았을 때 감옥 텔레포트 적용 등의 **주요 게임 규칙들을 집행**하는 심판 역할을 수행합니다.
* **`/game/items/index.js`**
  - 서버 측 아이템 지급 데이터 로직입니다.
  - `ITEMS_BY_TEAM`이라는 데이터 풀을 활용해 공격팀과 방어팀을 가려내고 고정 하드코딩이 아닌 논리적인 분배 방식에 따라 초기 아이템을 랜덤하게 지급합니다.
